Experience Trails
Pre-Purchase Visibility Model
(Board Demo Build → April Pilot Build)
1. Objective
Balance conversion clarity with “insider premium.”
Pre-purchase should sell the vibe and format — not reveal the stops.

2. Visibility by Surface
A. Landing Page (Explore View)
Visible:
Trail name
Creator name + avatar
Neighborhood (general area)
Duration of stops
Price
1-line vibe descriptor
Hero image
Hidden:
Exact stop names
Addresses
Detailed insider hooks

B. Trail Detail Page (Before Purchase)
Visible:
Trail hero image
Creator snippet
Narrative overview paragraph
Neighborhood map (blurred/generalized)
Duration
Best time of day
Estimated budget range
Group-friendly badge
Stop Preview Structure:
Stop 1 → 1-line teaser only
Stop 2 → 1-line teaser only
Stop 3 → 1-line teaser only
Final Stop 4 → 1-line teaser only
Locked visual treatment


Hidden:
Stop names
Exact locations
Full descriptions

C. Checkout Page
Visible:
Trail name
Date selector
Group size selector
Total price
“What you get” bullet list:
3–4 curated stops
Progressive unlock
Completion reward
Creator commentary
Hidden:
Stop names
Addresses

D. Post-Purchase Experience
Upon “Start Trail”:
Stop 1 fully revealed (name, address, photo, creator notes)
Check-in mechanic
Stop 2 unlocks after check-in
Completion badge at end

3. Board Build vs Pilot Build
For March 9:
Use single visibility model (no A/B)
1–2 fully demo-ready trails
Remaining trails structured but lighter depth
For April Pilot:
A/B test:
Show Stop 1 name pre-purchase vs hide all
Price bands
Instrument event tracking
